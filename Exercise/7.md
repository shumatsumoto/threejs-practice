## 1. 問題タイトル

7 本目：オブジェクトのスケール変更

## 2. 学習目標

この演習を通じて、以下の技術を習得します。

- オブジェクトの拡大・縮小（Scale）の操作
- ローカル座標系での変形

## 3. 新しい概念の解説

**Scale（スケール）**
オブジェクトの拡大縮小率を定義するプロパティです。
初期値は `(1, 1, 1)` です。
`2` にすると2倍、`0.5` にすると半分になります。
`cube.scale.set(x, y, z)` で一括設定も可能です。

```javascript
// X方向に2倍、Y方向に0.5倍にする
cube.scale.set(2, 0.5, 1);

// 個別に設定する場合
cube.scale.x = 2;
```

## 4. 課題の説明

キューブの形を変形させて、横に長い「板」のような形状にしてください。

## 5. 必須要件

以下の機能を実装してください。

- **スケールの変更**:
  キューブの `scale` プロパティを操作します。

- **変形**:
  X軸（幅）を `2`倍、Y軸（高さ）を `0.5`倍、Z軸（奥行き）を `1`倍にします。
  `cube.scale.set(2, 0.5, 1);` が便利です。

- **確認**:
  正立方体だったキューブが、横長の平たい板のような形に変形していることを確認します。

## 6. ヒント

- **ヒント 1（スケールの初期値）**:
  他のプロパティと違い、スケールの初期値はすべて `1` です。
  `0` にしてしまうと、大きさがゼロになり見えなくなってしまいます。
- **ヒント 2（個別設定）**:
  `cube.scale.x = 2;`
  `cube.scale.y = 0.5;`
  のように軸ごとに設定します。
- **ヒント 3（一括設定）**:
  `cube.scale.set(2, 0.5, 1);`
  を使うと一行で書けます。変更しない軸（Z）も明示的に `1` を指定するのを忘れずに。
- **ヒント 4（ローカル座標）**:
  スケールを変更すると、そのオブジェクトが持つ「ローカル座標系」も歪みます。
  この状態で子要素を追加すると、子要素も一緒に歪んでしまう点に注意が必要です。
- **ヒント 5（ジオメトリとの違い）**:
  `BoxGeometry(2, 0.5, 1)` で最初からその形で作るのと、`scale` で変形させるのは、見た目は同じですが内部処理が異なります。
  アニメーションで伸び縮みさせるなら `scale` が適しています。

## 8. 模範解答

```javascript
import * as THREE from "three";

const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(
  75,
  window.innerWidth / window.innerHeight,
  0.1,
  1000
);
const renderer = new THREE.WebGLRenderer();
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);

const geometry = new THREE.BoxGeometry(1, 1, 1);
const material = new THREE.MeshBasicMaterial({ color: 0x00ff00 });
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);

camera.position.z = 5;

// スケールの変更
// 幅を2倍
cube.scale.x = 2;
// 高さを半分
cube.scale.y = 0.5;

renderer.render(scene, camera);
```
